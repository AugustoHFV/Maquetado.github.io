Conclusiones:
Para desplegar una página en GitHub Pages no se necesita configurar servidorese, gestionar bases de datos ni pagar un hosting. Es una forma rápida de obtener una URL pública.
El proceso se puede hacer creando un repositorio o habilitando Pages en la configuración de cualquier repositorio existente. Funciona seleccionando la rama desde donde se sirven los archivos.
git push permite una automatización completa, actualizando el sitio automáticamente.
Tener proyectos en GitHub permite ver código y el demo en vivo simultáneamente. Al ser sitios estáticos, son inmunes a las vulnerabilidades de bases de datos o inyecciones de SQL.
